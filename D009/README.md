# 스마트 제조 데이터 정제·집계 보고서

## 1. 데이터 개요
- 출처: 스마트 제조 설비 IoT 센서 로그
- 기간: 2024-01-01 ~ 2024-03-10 (약 2개월)
- 원본 행 수 / 정제 후 행 수: 100,020행 → 98,010행

## 2. 발견한 품질 문제
- 결측: Temperature_C 1%, Predictive_Maintenance_Score 1.8%
- 중복: 20건
- 이상치: Vibration_Hz 음수 값, Power_Consumption_Kw 극단값
- 표기 혼재: Operation_Mode에 공백·대소문자 표기 혼재

## 3. 처리 결정과 근거 (5줄로)
1. 중복 행 20건 → 제거 (동일 키·동일 값이 반복되는 것은 시스템 중복 입력 오류로 간주)
2. Operation_Mode ' Active ', 'active', 'Active', 'ACTIVE' → 'Active' (동일한 작동 모드)
3. Vibration_Hz 음수 값→ 제거 (진동은 물리적으로 음수가 될 수 없는 값이므로 센서/기록 오류로 판단)
4. Power_Consumption_kW 극단값 → 상위 1% 컷오프 (정상 범위를 크게 벗어난 값으로 센서 오류로 추정)
5. Temperature_C 결측 1000건 → 제거 (기계 간 비율 차이가 거의 없어 MCAR로 판단, 전체 비율도 1%로 작아 제거해도 편향 위험 낮다고 판단)
6. Predictive_Maintenance_Score 결측 1769건 → 기계별 중앙값 대체 + 결측 플래그 (특정 기계(5,17,33)에 결측이 몰려있어 MAR로 판단, 그룹별 대체로 정보 손실 최소화)

## 4. 주요 KPI 결과 (2줄)그 월:
- High → Low로 갈수록 생산속도는 감소, 오류율은 증가한다. 반면, 불량률은 세 등급 모두 유사하다. → 효율 등급을 가르는 핵심 변수는 오류율과 생산속도이며, 품질검사 불량률과는 무관하다.
- 50대 기계 중 저효율 비중이 유독 높은 기계가 없다. → 효율 저하가 특정 설비의 결함이 아니라 공정 자체의 문제일 가능성이 있다. 

## 5. 한계와 후속 작업
- 효율 등급에 영향을 주는 다른 요인(시간대, 다른 센서값 등)에 대한 추가 분석이 필요하다.
- Power_Consumption_kW 극단값이 실제 센서 오류인지, 특정 조건에서 발생하는 정상 현상인지 추가 검증이 필요하다.
- Predictive_Maintenance_Score 결측이 특정 기계에 집중된 원인에 대한 확인이 필요하다. (그룹별 중앙값 대체가 해당 기계들의 실제 설비 정비 시급성을 과소평가 할 위험이 있음)
